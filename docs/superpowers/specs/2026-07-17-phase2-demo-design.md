# Phase 2 Demo UI 设计说明

## 1. 范围

本 demo 只覆盖已经确认的二期需求：

- 规格树生成
- 条款新旧比较增强
- 使用量记录，以及基于 IP 的工厂识别和费用分摊

本 demo 不引入登录、个人用户管理、Prompt 设置，也不做条款比较的全局任务中心。

## 2. 导航

侧边栏只展示：

- `スペックツリー`
- `スペック新旧比較`
- `利用実績`

`プロンプト設定` 不在客户演示范围内，导航中隐藏。

## 3. 规格树

### 3.1 页面结构

`スペックツリー` 内部使用两个页签：

- `ツリー生成`
- `タスク管理`

这里的任务管理只服务于规格树，不是全系统的任务中心。

### 3.2 ツリー生成

左侧保留一期的界面风格和操作习惯：

- 支持一次上传一批 PDF/TIF 文件。
- 保留一期的清空按钮。
- 保留可滚动的上传文件列表。
- 保留单个文件删除 `×`。
- 上传完成后，直接在上传文件列表中指定 root 文件。
- 不再单独提供 root 下拉选择区域。
- 每个被指定为 root 的文件，对应生成一棵独立规格树任务。

文件行的交互：

- 未选中 root 时显示 `＋`。
- 已选中 root 时显示 `ツリー`，并高亮该行。
- 点击该标记即可切换 root 状态。

左侧底部保留两个 Excel 输入：

- `② 補完リスト Excel`
- `③ 削除リスト Excel`

每个 Excel 块显示当前文件，并提供 `変更` 上传入口。

### 3.3 右侧结果区

右侧直接显示生成后的规格树结果。

一次生成多棵树时，右侧结果区使用树结果 tab 展示，一棵树一个 tab。

工具栏沿用一期规则：

- 放大
- 缩小
- 重置
- TD / LR 方向切换
- 一个 `ダウンロード` 下拉按钮，包含：
  - Excel
  - CSV
  - MMD
  - SVG

### 3.4 タスク管理

任务列表采用类似下载器的任务行形式。一个 root 对应一条任务。

状态 tab：

- `すべて`
- `待機中`
- `実行中`
- `完了`
- `失敗 / 中止`

每条任务展示：

- 状态
- root 文件名
- 工厂 / IP
- 进度
- 全局队列位置，例如 `全体キュー 3番目`
- 保存期限，例如 `保存期限：残り44日`
- 根据状态展示操作：
  - `中断`
  - `リトライ`
  - `結果表示`
  - 单个下载

任务完成后出现在 `完了` 中。

已完成任务支持多选和批量下载：

- `選択した結果を一括ダウンロード`
  - Excel
  - CSV
  - MMD
  - SVG

点击 `結果表示` 会返回 `ツリー生成`，并加载该任务对应的树结果。

## 4. 基于 IP 的任务可见性

本 demo 按 IP / 工厂作为任务可见性单位。

规则：

- 服务器写死 IP 白名单。
- 来源 IP 不在白名单时，用户不能使用系统。
- 来源 IP 在白名单时，系统将 IP 映射到工厂。
- 规格树任务归属于来源 IP / 工厂。
- 规格树的任务管理只展示当前 IP / 工厂的任务。
- 系统不区分个人用户，也不区分浏览器会话。

这与费用分摊模型一致：

- 任务归属：IP / 工厂
- 任务可见性：当前 IP / 工厂
- 使用量统计：IP / 工厂

如果多台电脑共享同一个出口 IP，则视为同一工厂用户组。

## 5. 条款新旧比较

条款新旧比较仍保留在功能页面中，不进入规格树任务管理列表。

二期确认的展示需求：

- 支持新增 / 删除条款导致项目号顺延或逆延的情况。
- 变更分类扩展为：
  - `変更`
  - `追加`
  - `削除`
  - `変更なし`
- 对已对齐的变更条款展示句子 / 字句级高亮：
  - 删除文字：红色 + 删除线
  - 新增文字：绿色
- Excel 导出保留同样的高亮规则。
- 条款内容支持英文到日文翻译。

## 6. 利用実績

原系统日志页改为客户可理解的 `利用実績`。

该页聚焦使用量和费用分摊，不展示工程维护操作。

展示内容：

- 当前工厂 / IP
- 使用件数
- 使用页数
- 工厂 / IP 别汇总
- 使用明细列表
- CSV 导出

隐藏工程操作：

- 缓存清理
- 手动 cleanup

使用明细记录：

- 操作时间
- 工厂
- IP
- 功能
- 处理件数或页数
- 操作类型
- 备注

## 7. 数据保存期限和缓存复用

任务列表可以展示保存期限内的已完成结果。

保存期限规则：

- 已完成任务在保存期限内可以查看结果。
- 过期结果显示 `保存期限切れ`。
- 过期结果的查看和下载操作禁用。

缓存复用是内部优化：

- 可以用很轻的标签展示，例如 `既存結果利用`。
- 不作为用户可管理的历史或缓存功能。
- 不提供缓存搜索、缓存删除、手动选择缓存等 UI。

## 8. 非目标

本 demo 不包含：

- 登录
- 个人身份
- 用户角色
- 按个人隔离任务
- 除 IP / 工厂可见性之外的工厂级共享控制
- 客户可见的 Prompt 管理
- 条款比较任务管理
- 全局混合功能任务中心
