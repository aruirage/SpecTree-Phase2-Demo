# LLM OCR 二期 UI 优化设计方案

**版本：** v1.0  
**基线：** 现有 Demo（深色侧栏 + 左配置 / 右结果双栏布局）  
**原则：** 交互怎么方便怎么来、实现怎么简单怎么来

---

## 1. 现状与问题

### 1.1 已有 UI 资产

| 模块 | 现状 |
|------|------|
| 布局 | 240px 深色侧栏 + 主内容区，可折叠 |
| 规格树 | 左：上传/Root/生成；右：Mermaid 树 + 缩放 |
| 条款对比 | 左：旧/新 PDF + 执行；右：条项表格 + 高亮 + 全屏 |
| 导航 | 仅 2 项：スペックツリー / スペック新旧比較 |
| 品牌色 | `#0076BF`（大金蓝） |

### 1.2 待解决问题

| 问题 | 影响 |
|------|------|
| 二期功能无入口 | 任务列表、Prompt、License、Excel 登记无处放 |
| 规格树缺 Excel 区 | 补全/删除 Excel 无 UI |
| 无全局任务视图 | 多树并发、排队、停止/再执行无统一列表 |
| 状态反馈分散 | 进度、过期、缓存命中各自为政 |
| 单页逻辑过重 | SpecTree / ClauseComparison 各 1000+ 行，难维护 |
| Session 心智 | 页面绑 sessionId，与 Job 历史不兼容 |

---

## 2. 设计目标

1. **少页面**：4 个路由 + 1 个弹窗，不做独立管理大页
2. **少点击**：Excel 默认用「当前文件」；日常只传 PDF + 选 Root
3. **一眼可见**：License 用量、进行中任务数放侧栏 Widget
4. **延续现有风格**：不推翻 Demo，在其上增量扩展
5. **组件可复用**：上传区、进度条、Banner、任务行统一组件

## 7. 页面设计

### 7.1 スペックツリー `/spec-tree`

- ① PDF/TIF 上传（多文件）
- ② 补全 Excel：当前文件 + 更换
- ③ 删除 Excel：当前文件 + 更换
- Root 指定 → 生成を開始
- 右：Mermaid 画布

**进度四阶段：** PDF/TIF 解析 → LLM 生成 → 补全 Excel → 删除 Excel

### 7.2 スペック新旧比較 `/clause-comparison`

- 旧/新 PDF 上传 → 比較を実行
- 四分类过滤 + 字句高亮 + 日语译文
- 进度三阶段：extract → cross_clause → translate

### 7.3 タスク一覧 `/tasks`

- 全宽表格，规格树/对比混合
- 操作：查看 / 停止 / 再実行 / 取消排队

### 7.4 プロンプト設定 `/prompts`

- 两个 Tab：规格树（1 卡片）/ 对比（3 卡片）
- 保存 / 恢复默认

### 7.5 License Widget + Modal

- Widget：本月页数 / 配额 / 进度条
- Modal：上传 License、CSV 导出、审计导出
